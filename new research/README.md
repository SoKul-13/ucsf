# AI-READI Cognitive Impairment Analysis: Replication & Validation

This repository contains scripts for replicating and validating findings related to diabetes severity (CGM vs. HbA1c) and cognitive function (MoCA scores).

## Order of Execution

To fully replicate the setup and run the validation scripts from the walkthrough, run the scripts in this order:

1. **`extract_data.py`**
   - **What it does**: Parses the raw AI-READI clinical data (`participants.tsv`, `measurement.csv`, `condition_occurrence.csv`, `observation.csv`) and the Dexcom G6 CGM JSON files. It extracts the necessary covariates (HbA1c, Age, BMI, Education, etc.) and CGM metrics (GMI, Mean Glucose, TIR).
   - **Expected Output**: A clean, unified CSV file named `master_cgm_moca_dataset.csv`. This file contains one row per patient, aggregating all clinical demographics, diagnostic flags, and continuous glucose variables, serving as the foundation for all subsequent scripts.

2. **`analyze_side_by_side.py`**
   - **What it does**: Loads the extracted dataset and runs the multivariable OLS and Logistic Regression models. It directly compares the predictive power of CGM-derived GMI against lab-derived HbA1c side-by-side.
   - **Expected Output**: A formatted markdown report named `report_side_by_side.md`. This report renders side-by-side tables containing the adjusted MoCA means and multivariable Odds Ratios for both the GMI and HbA1c models.

3. **`moca_validity.py`**
   - **What it does**: Evaluates the construct validity of the MoCA score in the dataset. It pulls patients with documented neurodegenerative or cognitive disease OMOP codes and runs a permutation bootstrap hypothesis test to check if their MoCA scores are significantly lower than those without conditions, stratified by age and diabetes status.
   - **Expected Output**: A markdown report named `moca_validity_results.md`. It explicitly lists out the patient counts, mean MoCA scores, and calculated p-values for both the overall dataset and every sub-stratification group.

4. **`export_exact_results.py`**
   - **What it does**: Runs the multivariable logistic regression models against both the total cognitive impairment (MoCA < 26) and specific disease incidences (Alzheimer's, Parkinson's, Multiple Sclerosis, etc.). It calculates precise Beta coefficients, standard errors, Odds Ratios, and exact p-values for every single covariate (Age, BMI, GMI/HbA1c).
   - **Expected Outputs (CSVs)**: 
     - `regression_results_total_moca.csv`: A structured spreadsheet containing the exact (unrounded) numerical outputs of the statistical models predicting total MoCA impairment.
     - `regression_results_specific_impairments.csv`: A structured spreadsheet containing the exact numerical outputs for the models predicting the incidence of specific cognitive and neurodegenerative diseases.

## Generated Reports Summary

| Report Output File | Generated By | Primary Purpose | How it is Useful / Comparison |
| :--- | :--- | :--- | :--- |
| **`report.md`** | `analyze_data.py` | Baseline statistical replication using GMI instead of HbA1c. | Establishes the foundation. Shows baseline demographics (Table 1), unadjusted/adjusted means (Table 2), and multivariable ORs (Table 3) strictly for GMI. |
| **`report_side_by_side.md`** | `analyze_side_by_side.py` | Direct multivariable comparison of GMI vs HbA1c side-by-side. | The most critical file for proving your thesis. Directly contrasts GMI and HbA1c within the same population to prove GMI's superiority in predicting impairment. |
| **`moca_validity_results.md`** | `moca_validity.py` | Compares cohort definition methods (AI-READI Strings vs Exhaustive OMOP). | Validates your data extraction methodology. Proves that the `mhoccur` strings method yields a much larger, statistically robust sample (N=242) vs OMOP (N=9). |
| **`comparison_results.md`** | `gridsearch_ml.py` | Machine Learning GridSearch comparing predictive power of all metrics. | Shifts from statistical inference to prediction. Ranks GMI, HbA1c, Mean Glucose, and TIR by Random Forest AUC and Logistic Regression Accuracy. |
| **`regression_results_total_moca.csv`** <br> **`regression_results_specific_impairments.csv`** | `export_exact_results.py` | Raw regression coefficients, standard errors, p-values, and 95% CIs. | Best for raw data manipulation. Useful if you need to build custom Forest Plots in Python/R or need exact precision rather than formatted markdown tables. |
